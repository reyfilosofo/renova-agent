# LUCEM Verification Protocol

## Propósito

LUCEM es la capa de verificación del ecosistema ℛenova / SERESARTE / NOUS.

Su función es impedir que la producción acelerada de documentos, ideas, asistentes, libros o productos derive en afirmaciones sin fuente, contradicciones internas, promesas excesivas, errores conceptuales o riesgos reputacionales.

## Principios

1. Toda afirmación fuerte requiere fuente o marca de incertidumbre.
2. Toda hipótesis debe distinguirse de un hecho.
3. Toda cita debe ser verificable.
4. Toda promesa técnica debe ser conservadora.
5. Toda automatización debe reconocer gobernanza humana.
6. Toda versión debe dejar rastro.
7. Toda publicación debe poder auditarse.

## Matriz de revisión

| Dimensión | Pregunta | Riesgo si falla |
|---|---|---|
| Fuente | ¿La afirmación tiene fuente verificable? | Alucinación documental |
| Concepto | ¿El concepto respeta ℛenova? | Deriva conceptual |
| Técnica | ¿La promesa técnica es realista? | Sobreventa tecnológica |
| Ética | ¿Hay límites claros? | Riesgo de daño o manipulación |
| Marca | ¿Respeta SERESARTE/NOUS? | Pérdida de identidad |
| Archivo | ¿La versión está documentada? | Pérdida de linaje |
| Decisión | ¿El entregable ayuda a decidir? | Retórica sin utilidad |

## Etiquetas de riesgo

- `LUCEM-OK`: revisado y aceptado.
- `SOURCE-NEEDED`: requiere fuente.
- `CLAIM-RISK`: afirmación excesiva.
- `CONCEPT-DRIFT`: deriva conceptual.
- `TECH-OVERCLAIM`: promesa técnica no sustentada.
- `ETHICAL-RISK`: riesgo ético o reputacional.
- `PRIVATE-DATA`: posible dato sensible.
- `REWRITE-REQUIRED`: requiere reescritura.

## Checklist mínimo antes de publicar

- ¿Hay afirmaciones científicas? Si sí, ¿tienen fuente?
- ¿Hay promesas de autonomía? Si sí, ¿están limitadas?
- ¿Se menciona IA? Si sí, ¿se evita hype genérico?
- ¿El texto conserva diferencia entre ℛenova, SERESARTE, NOUS, LUCEM y Press?
- ¿El entregable tiene propósito claro?
- ¿Se puede rastrear de qué versión viene?
- ¿Hay datos privados o claves?
- ¿El lenguaje es publicable?
