# SERESARTE Brain — punto de arranque

Este documento inaugura la capa de segundo cerebro dentro del repositorio `renova-agent`.

## Propósito

Convertir este repositorio en una bóveda operativa de conocimiento para:

- SERESARTE;
- ℛenova;
- NOUS;
- DAO IAO;
- Renova Press;
- Rey Filósofo;
- clientes y proyectos estratégicos;
- libros, dossiers, brandbooks, prompts y entregables.

## Regla central

ChatGPT debe funcionar como capa filosófica, editorial y estratégica. Codex debe funcionar como capa operativa sobre archivos reales: clasificar, enlazar, crear índices, mantener memoria documental, generar plantillas y registrar cambios.

## Primer flujo de uso

1. Depositar materiales en `00_INBOX/`.
2. Pedir a Codex que actualice `MANIFEST.md`.
3. Clasificar cada archivo en la carpeta correspondiente.
4. Agregar front matter YAML a notas procesadas.
5. Crear enlaces internos estilo Obsidian.
6. Registrar decisiones en `07_DECISION_LOG/`.
7. Guardar entregables finales en `08_OUTPUTS/`.

## Archivos centrales

- `AGENTS.md`: constitución operativa para Codex.
- `MANIFEST.md`: inventario vivo del repositorio.
- `09_META/MAPA_MAESTRO.md`: mapa del sistema.
- `09_META/taxonomia.md`: clasificación general.
- `09_META/glosario_maestro.md`: vocabulario conceptual.
- `09_META/prompt_maestro_codex.md`: instrucción principal para Codex.
- `09_META/prompt_maestro_chatgpt.md`: instrucción principal para ChatGPT.

## Próxima acción recomendada para Codex

Ejecutar una auditoría inicial del repositorio:

```text
Revisa este repositorio como bóveda SERESARTE Brain. Lee AGENTS.md, actualiza MANIFEST.md, detecta carpetas faltantes, clasifica archivos existentes y propone los primeros backlinks conceptuales sin borrar ni sobrescribir originales.
```
