# Gobernanza editorial y tecnica

Este documento define reglas minimas para sostener calidad, continuidad y seguridad en los proyectos web de SERESARTE.

## Autoridad del sistema

GitHub es el archivo maestro. Las plataformas de despliegue publican copias vivas, pero la version gobernante es el repositorio.

## Criterios de calidad

Cada landing o pieza debe cumplir:

- Claridad de proposito.
- Identidad visual coherente.
- Texto revisado.
- Jerarquia editorial.
- Navegacion legible.
- Buen rendimiento movil.
- Ausencia de informacion privada.
- Registro de version.

## Revision editorial

Antes de publicar se revisa:

- Ortografia.
- Sintaxis.
- Tono institucional.
- Consistencia SERESARTE.
- Call to action.
- Riesgos legales o reputacionales.
- Creditos y derechos de imagen cuando aplique.

## Revision tecnica

Antes de publicar se revisa:

- Carga en navegador.
- Responsive.
- Enlaces activos.
- Imagenes optimizadas.
- No hay errores en consola.
- HTML semantico.
- Accesibilidad basica.

## Nomenclatura de carpetas

Usar minusculas y guiones:

- `fundacion-seresarte`
- `seresarte-media`
- `entra-al-lienzo`
- `mural-colectivo`
- `natura-movil`
- `renova-press`

## Versionado editorial

- `v0.1`: idea o prototipo inicial.
- `v0.5`: prototipo navegable.
- `v0.9`: listo para revision final.
- `v1.0`: primera version publicada.
- `v1.1`: correccion menor.
- `v2.0`: redireccion o rediseño mayor.

## Politica de datos

No subir documentos legales, datos personales, datos bancarios, expedientes medicos, contratos privados o informacion de menores al sitio publico. Si se necesita referencia interna, debe existir una version publica depurada.
